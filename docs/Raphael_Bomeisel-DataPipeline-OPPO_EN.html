The Current Challenge

  • Receiving sales and inventory data in decentralized files (.txt).
  • Lack of a central system to process, store, and consolidate information.
  • Difficulty in extracting fast and reliable insights for decision-making.

The Opportunity

  • Building a solid and scalable database from scratch.
  • Empowering leadership with accurate, real-time dashboards.
  • Transforming raw data into competitive intelligence to optimize sales and inventory.

Our Proposal: A Modern Data Pipeline

Fast

Implementation of the complete structure and delivery of the first dashboards in 2 to 3 weeks.

Cost-Effective

Using Google Cloud's serverless services, at no cost for the first year.

Scalable

A scalable architecture ready to support data volume growth without needing a redesign.

Phase 1: Ingestion and Storage

Objective

Create a central, secure, and organized repository to receive raw sales and inventory files.


Deliverables

  • Configured Cloud Storage bucket.
  • Defined folder structure for file versioning.
  • First files uploaded and validated.

Estimated Duration: 1 business day

Phase 2: Processing and Modeling

Objective

Transform raw data into clean, structured, and enriched information with business metrics.


Deliverables

  • BigQuery tables for raw and processed data.
  • SQL scripts for cleaning, integration, and KPI calculation (e.g., Sell-Through).
  • Final data model, ready for analysis.

Estimated Duration: 5-7 business days

Phase 3: Visualization and Insights

Objective

Connect the processed data to a BI tool to create interactive and accessible dashboards for the team.


Deliverables

  • Secure connection between BigQuery and Looker Studio.
  • Published dashboards with sales and inventory analysis.
  • Defined user access governance.

Estimated Duration: 3-4 business days
